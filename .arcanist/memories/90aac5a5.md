In `on_ready`, background task loops (`daily_digest.start()`, `reminder_loop.start()`, `eod_reminder_loop.start()`) must be started BEFORE any `await` calls that could raise exceptions (such as `bot.tree.sync()`). If `bot.tree.sync()` is called first without a try/except and it raises (e.g. Discord rate-limits the bot on restart), `on_ready` aborts immediately and the loops are never started — silently killing all background functionality. The fix: start all loops first (guarded with `is_running()` checks), then wrap both the global sync and per-guild syncs in individual try/except blocks so failures are logged as warnings rather than crashing `on_ready`.
