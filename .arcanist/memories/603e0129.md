In `post_daily_digest` (and anywhere else bot channel/member lookups are performed), `bot.get_channel()` and `channel.guild.get_member()` only check the local in-memory cache. If the bot starts without guild activity to warm the cache, these return `None` and the original code silently hit `continue` with no log output — making failures completely invisible. The correct pattern is to always fall back to the API calls `await bot.fetch_channel()` and `await channel.guild.fetch_member()` on cache misses, and to log a `WARNING` on every cache miss and an `ERROR` if the API call itself fails so digest skips are always traceable in logs.
