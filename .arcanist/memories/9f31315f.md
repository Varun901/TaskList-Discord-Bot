When a periodic task (like a daily digest) retries after partial failure, it can send duplicate messages to users who already received theirs. Maintain a per-recipient tracking set (e.g., `_digest_sent_today`) that records which recipients were successfully delivered to. On retry, skip already-delivered recipients. Clear this tracking set once per calendar day, not on every retry attempt. This is separate from the top-level 'did the whole job complete' flag — you need both: a coarse flag to know when to stop retrying entirely, and a fine-grained set to avoid duplicates during retries.
